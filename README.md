Enterprise Code Design Skill

enterprise-code-design 是一个用于复杂编码任务的 Codex Skill，目标是让 AI 在写代码前先理解项目架构、设计方案、评估影响范围，并在用户确认后再执行实现。

该 Skill 适合企业级或高风险开发场景，例如新增复杂业务功能、跨模块改造、接口变更、数据库变更、核心链路调整、性能优化、安全改造、重构和迁移任务。

核心能力

- 先调研当前项目架构、技术栈、设计模式、命名习惯和测试方式。
- 编码前先输出中文设计方案，并等待用户确认。
- 按风险等级决定设计、测试、发布和文档深度。
- 明确评估时间成本、复杂度、可扩展性、性能和影响范围。
- 修改代码时保持最小影响范围，不碰无关代码、注释、格式、配置或文档。
- 新增或修改代码必须添加中文注释。
- 强制考虑参数校验、异常处理、日志、配置抽取、安全性和数据一致性。
- 实现完成后补充测试用例，或明确说明测试方式。
- 交付前执行可行的验证命令。
- 测试完成后生成中文 Markdown 复盘文档。

工作流程

1. 判断任务风险等级。
2. 查看当前项目架构和代码风格。
3. 输出中文设计方案。
4. 说明方案评估维度和影响范围。
5. 等待用户确认。
6. 按确认后的方案编码。
7. 编写测试或说明测试方式。
8. 执行验证命令。
9. 生成任务复盘文档。

适用场景

- 复杂业务功能开发
- 多模块代码改造
- API 或数据库变更
- 核心链路调整
- 并发、性能或安全问题处理
- 系统重构或技术迁移
- 需要设计评审、测试验证和文档沉淀的任务

复盘文档规范

测试完成后，Skill 会要求生成一份中文 Markdown 复盘文档，用于记录完整开发过程、设计方案、测试方式、结果查看方法、优点不足和后续风险。

文档命名格式：

    yyyyMMdd（做了XXX）任务记录.md

示例：

    20260609（新增RTCM解码）任务记录.md
    20260609（修复订单回调幂等）文档.md

使用方式

在复杂开发任务中，可以这样要求 Codex：

    使用 enterprise-code-design 流程，先看项目架构，再给我设计方案，确认后再编码。

Codex 会先进行项目调研和方案设计，待用户确认后再开始修改代码。
